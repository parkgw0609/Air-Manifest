MAWB Status

          

            
타입 Select

            
MAWB의 EDI 전송 단계를 필터링합니다. 기본값은 All이며 선택 가능한 값은 All / New / EPNINF Sent / FWB/FHL Sent입니다.

          

        


        

          
MAWB No.

          

            
타입 Multi-input 상호배타

            
MAWB 번호를 직접 입력하여 검색합니다. + 버튼으로 복수 입력이 가능합니다.

            
이 필드에 값이 있으면 HAWB No.와 ETD 필드가 자동 비활성화됩니다.

          

          

            
속성값

            
              
필드 IDs-mawb-no

              
이벤트oninput → onMawbInput()

              
배타 처리HAWB No(sf-hawb), ETD(sf-etd) disabled 토글

            
          

        


        

          
HAWB No.

          

            
타입 Multi-input 상호배타

            
HAWB 번호로 검색합니다. 입력 시 연관된 MAWB를 역방향 조회합니다.

            
이 필드에 값이 있으면 MAWB No.와 ETD 필드가 자동 비활성화됩니다.

          

          

            
속성값

            
              
필드 IDs-hawb-no

              
이벤트oninput → onHawbInput()

              
배타 처리MAWB No(mawb-input-wrap 부모 .sf), ETD(sf-etd) disabled 토글

            
          

        


        

          
Carrier

          

            
타입 Lookup Input

            
Carrier 코드 또는 명칭을 직접 입력하거나, 🔍 아이콘 클릭 시 Carrier 검색 팝업(cpop-overlay)이 열립니다. 팝업에서 선택 시 이 필드에 값이 자동 반영됩니다.

          

          

            
속성값

            
              
필드 IDs-carrier

              
팝업 함수openCarrierPopup()

              
검색 로직carrier 코드 LIKE + carrierName LIKE (대소문자 무시)

            
          

        


        

          
ETD (출발 예정일)

          

            
타입 DateRange 상호배타

            
항공편 출발 예정일 범위로 필터링합니다. 기본값은 조회 기준일 기준 ±2주. MAWB No. 또는 HAWB No. 입력 시 비활성화됩니다.

          

          

            
속성값

            
              
From 필드 IDs-etd-from

              
To 필드 IDs-etd-to

              
기본값from: 오늘 -14일, to: 오늘 +14일

              
포맷YYYY-MM-DD (input[type=date])

            
          

        


        

          
EDI Send Status

          

            
타입 Select

            
EDI 전문 전송 처리 상태로 필터링합니다. 옵션: All / N/A / Processing / Complete / Error

          

          

            
값설명

            
              
N/A전송 이력 없음 (초기 상태)

              
ProcessingKTNET 전송 요청 후 응답 대기 중

              
Complete전송 성공 확인

              
Error전송 실패 — Need Resend 자동 Yes

            
          

        


        

          
Need Resend

          

            
타입 Select

            
재전송 필요 여부로 필터링합니다. EDI Send Status가 Error일 때 자동으로 Yes가 설정됩니다. DB 저장값은 Need Resend / Normal이며 화면에는 Yes / No로 표시됩니다.

          

          

            
화면값DB값

            
              
YesNeed Resend

              
NoNormal

            
          

        


        

          
초기화 / 재조회 버튼

          

            
↺ 초기화

            
모든 검색 조건을 기본값으로 리셋하고 즉시 재조회합니다. MAWB/HAWB No. 입력으로 비활성화된 필드도 함께 복원됩니다.

          

          

            
∧ 접기/펼치기

            
검색 필터 영역을 접거나 펼칩니다. 접힌 상태에서도 현재 조건은 유지되며 Search 버튼 조회는 정상 동작합니다.

          

        

        

      


      
      


        
        

          

        


        
        

        

          
MAWB 그리드 버튼 규칙


          

            
EPNINF Send EDI

            

              
                
활성 조건체크된 MAWB 1건 이상 & Status = New OR Need Resend = Yes

                
비활성 조건선택 없음 / 이미 Processing 상태 포함

                
클릭 동작선택 건 ediStatus → Processing, status → EPNINF Sent 즉시 변경

                
비동기 결과KTNET 응답 후 Complete 또는 Error로 업데이트

                
다건 처리체크된 모든 건 일괄 전송 지원

                
함수epninfSend()

              
            

          


          

            
FWB/FHL Send EDI

            

              
                
활성 조건체크된 MAWB 1건 이상 & EPNINF Sent 상태 이상

                
비활성 조건EPNINF 미전송 건 포함 / Processing 상태 건 포함

                
클릭 동작FWB(Master) + FHL(House) 전문 동시 전송. status → FWB/FHL Sent

                
선택 제한다건 일괄 처리 가능

                
함수fwbFhlSend()

              
            

          


          

            
MAWB History 팝업

            

              
                
활성 조건MAWB 1건만 체크 선택

                
비활성 조건선택 없음 / 2건 이상 선택 → 오류 토스트

                
클릭 동작mpop-overlay 팝업 오픈, 전송 이력 드롭다운 구성

                
함수mawbHistory()

파일명MAWB_Info_YYYY-MM-DD.csv

화면 목적

          

            
한국 세관 항공 화물 EDI 전문(EPNINF / FWB / FHL)을 전송하는 화면. MAWB · HAWB 단위로 전송 상태를 관리하며 KTNET / uLogisHub 게이트웨이를 통해 실전문을 발송합니다.

          

        


        

          
화면 구성 (4-Layer)

          

            
1

            
검색 필터 영역
MAWB No, HAWB No, Carrier, ETD 범위, EDI Send Status, Need Resend 조건으로 조회. ∧ 버튼으로 접기/펼치기.

          

          

            
2

            
MAWB Info. 그리드
조회된 MAWB 목록. 행 선택 → HAWB 그리드 자동 연계 조회.

          

          

            
3

            
HAWB Info. 그리드
선택된 MAWB에 속한 HAWB 목록. 행 선택 → 수출신고 정보 연계.

          

          

            
4

            
Export Declaration Info. (Read Only)
HBL Detail > Others 탭 연동. 선택 HAWB의 수출신고 정보 조회 전용.

MAWB 그리드 버튼

          

            
MAWB History 팝업

            
선택 MAWB(단건)의 과거 EDI 전송 이력 조회. 전송 시점별 스냅샷을 Read Only로 확인.

          

          

            
EPNINF Send EDI 전송

            
선택 MAWB에 EPNINF 전문 전송. 
조건: Status = New 또는 Need Resend = Yes
결과: Status → EPNINF Sent, EDI Send Status → Processing → Complete/Error

          

          

            
FWB/FHL Send EDI 전송

            
FWB(Master AWB) + FHL(House AWB) 동시 전송.
조건: Status = EPNINF Sent 이상 (미전송 시 비활성)
다건 선택 일괄 전송 지원.

          

          

            
⬇ 엑셀 다운로드 CSV

            
현재 그리드 조회 데이터를 CSV 파일로 내보내기.

          

          

            
⚙ 컬럼 설정 다이얼로그

            
표시 컬럼 체크박스 선택 + 드래그로 순서 변경. 초기화 버튼으로 기본값 복원.

          

        


        

          
HAWB 그리드 버튼

          

            
HAWB History 팝업

            
선택 HAWB(단건)의 과거 전송 이력. MAWB + HAWB + 수출신고 정보 포함.

          

          

            
Amend Manifest 수정 전송

            
기 전송된 Manifest 수정 후 재전송. FWB/FHL 전송 완료 이후에만 활성화. 단건 선택.

          

          

            
Address Amend 수정 전송

            
Shipper/Consignee/Notify 주소 정보만 수정 전송하는 경량 수정 기능. 단건 선택.

검색 필터 영역 토글. 접힌 상태에서도 조회 기능 유지.

          

          

            
↺ (초기화)

            
검색 조건 전체 초기화 후 기본 조건으로 재조회.

컬럼 설정 다이얼로그

          

            
호출 조건

            
MAWB / HAWB / Carrier 그리드 우측 ⚙ 버튼 클릭

          

          

            
기능

            
    
              
  • 체크박스로 표시 컬럼 선택/해제
    • 
              
  • 드래그 앤 드롭으로 컬럼 순서 변경
    • 
              
  • 초기화: 기본 컬럼 순서로 리셋
    • 
              
  • 적용: 변경사항 즉시 그리드에 반영
    • 
              
  • 닫기 / X / 배경 클릭: 변경사항 적용 후 닫힘
    • 
            

          

        

        

      


      
      

        

          

        

        


        

          
EDI 전송 규칙

          
    
            
  • EPNINF 전송: Status = New 또는 Need Resend = Yes인 경우에만 활성화
    • 
            
  • FWB/FHL 전송: EPNINF Sent 이후에만 가능. 미전송 시 버튼 비활성
    • 
            
  • 전송은 다건 선택 일괄 처리 지원
    • 
            
  • 전송 후 즉시 Processing → KTNET 응답에 따라 Complete 또는 Error 비동기 업데이트
    • 
            
  • Error 발생 시 Need Resend = Yes 자동 설정
    • 
            
  • Processing 상태 건 포함 일괄 전송 시 오류 메시지 표시
    • 
          

        


        

          
화면 연동 규칙

          
    
            
  • MAWB 행 선택 → HAWB 그리드 자동 연계 조회 (미선택 시 빈 상태)
    • 
            
  • HAWB 행 선택 → Export Declaration Info. 자동 연계 표시
    • 
            
  • Export Declaration 영역은 Read Only — 수정은 HBL Detail > Others 탭에서만 가능
    • 
            
  • MAWB No. / HAWB No. 검색 필터는 상호 배타 (하나 입력 시 다른 필드 비활성)
    • 
            
  • MAWB No. / HAWB No. 입력 시 ETD 필터도 비활성
    • 
          

        


        

          
Amend 규칙

          
    
            
  • Amend Manifest: FWB/FHL 전송 완료 후에만 활성화, 단건만 처리
    • 
            
  • Address Amend: Shipper/Consignee/Notify 주소 필드만 수정 전송. 화물 정보 변경 불가. 단건
    • 
            
  • Amend 전송도 EDI Send Status 흐름 동일 적용
    • 
          

        


        

          
그리드 동작 규칙

          
    
            
  • MAWB 기본 정렬: ETD 오름차순 / HAWB 기본 정렬: HAWB No. 오름차순
    • 
            
  • OC Code1~3: 동일 코드 중복 입력 불가 (UI 단 Validation)
    • 
            
  • Cargo Description: 최대 20자 입력 제한
    • 
            
  • 컬럼 설정은 사용자별 저장 (서버 또는 LocalStorage — 구현 방식 협의 필요)
    • 
          

        


        

          
개발 협의 필요 사항

          

            

              ① 컬럼 설정 저장 방식: 서버 DB 저장 vs. LocalStorage

              ② EDI 비동기 응답: 폴링 주기 vs. WebSocket

              ③ 전송 오류 알림 방식: 토스트 vs. 팝업

              ④ 다건 전송 부분 성공 처리 방식 정의

              ⑤ Amend/Address Amend 화면 UX 상세 설계 필요
